# 暑期项目考核:商城系统

>传统MVC项目

## 开发设置及代码提交说明
```bash
用git add +文件名的方法 将发生改动的文件选中
用git commit说明改动内容
用git push将代码提交到github上
每次提交尽量围绕一个功能commit
如果同时在改动多个模块，可以分批提交
----------------------------------------------------------
前端文件放入 Front-end
后端文件放入 Back-end
sql保存文件 Sql-save
----------------------------------------------------------
变量命名规范 java及js中采用驼峰，sql中使用"_"字符
```
